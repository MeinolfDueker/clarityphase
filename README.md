# ClarityPhase 1.1.0 – Changelog

## Neu
- Enterprise-ready Erweiterung für **Custom Phases**
- Neue Helper-Funktionen für zentrale Phasenverwaltung
- Neue Enterprise-Einstellung **Custom Phases** im Plugin-Backend
- Custom Phases werden im Format `slug|Langer Name|Kurzer Name` gespeichert

## Erweitert
- Bestehende Standardphasen bleiben vollständig als Fallback erhalten
- Status-Auswahl nutzt jetzt bei Enterprise-Lizenz auf Wunsch eigene Phasen
- Phase-Anzeige und Phasen-Labels arbeiten jetzt mit dynamischen Phasen
- Filter, Quick Edit und Bulk Actions unterstützen ebenfalls Custom Phases
- Workflow-Schritte und Phase-Pills greifen auf die zentrale Phasenlogik zu

## Kompatibilität
- Keine bestehenden Funktionen entfernt
- Keine CSS-Anpassungen vorgenommen
- Bestehende Übersetzungszeilen nicht inhaltlich verändert
- Ohne Enterprise/ohne eigene Phasen läuft das Plugin weiter mit den bisherigen Standardphasen

## Version
- Vorher: **1.0.4**
- Jetzt: **1.1.0**
